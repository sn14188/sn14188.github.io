## Motivation

### 상태 로직 재사용의 한계

웹 애플리케이션에서 마우스 위치를 추적하는 로직이 있다고 가정해보겠습니다.
사용자가 화면 위로 커서를 움직이면, 컴포넌트는 현재 마우스의 좌표를 상태로 저장하고 이를 화면에 표시합니다.

```js
class MouseTracker extends React.Component {
  constructor(props) {
    super(props);
    this.handleMouseMove = this.handleMouseMove.bind(this);
    this.state = { x: 0, y: 0 };
  }

  handleMouseMove(event) {
    this.setState({
      x: event.clientX,
      y: event.clientY,
    });
  }

  render() {
    return (
      <div style={{ height: "100vh" }} onMouseMove={this.handleMouseMove}>
        <h1>Move the mouse around!</h1>
        <p>
          The current mouse position is ({this.state.x}, {this.state.y})
        </p>
      </div>
    );
  }
}
```

이제 마우스 주변에 고양이 이미지를 따라다니게 하고 싶습니다. 이를 위해서는 마우스 좌표를 `Cat` 컴포넌트에 전달해야 하는데, 다음과 같은 방법을 생각해볼 수 있습니다.

- `Cat` 컴포넌트를 마우스 컴포넌트 내부에서 직접 렌더링합니다
- 마우스 좌표를 `props`로 전달합니다
- 렌더링에 사용할 함수를 `props`로 전달하는 `render prop` 패턴을 사용합니다
- 고차 컴포넌트로 감싸서 사용합니다

그러나 이러한 방식들은 모두 마우스 추적 로직을 재사용하기 위해 컴포넌트 구조를 변경하거나, 새로운 컴포넌트를 추가로 만들어야 한다는 한계가 있습니다.
단순히 마우스 위치를 추적하는 상태 로직만 재사용하고 싶은데도 컴포넌트 계층 구조가 함께 복잡해지게 됩니다.

### 생명주기 중심 구조의 복잡성

클래스 컴포넌트는 생명주기를 기준으로 코드를 나누기 때문에 서로 관련 없는 코드가 한 메서드에 섞이고, 반대로 하나의 기능에 해당하는 코드가 여러 메서드에 흩어지게 됩니다.

- `componentDidMount`에 데이터 요청과 이벤트 리스너 등록이 함께 작성되면서, 서로 다른 기능이 하나의 생명주기 메서드에 뒤섞이게 됩니다
- 이벤트 리스너의 등록은 `componentDidMount`에, 정리는 `componentWillUnmount`에 작성되면서 하나의 기능이 여러 생명주기 메서드로 분산됩니다

이처럼 코드가 생명주기 기준으로 분리되면 코드의 흐름을 파악하기 어렵고 수정할 때에도 여러 메서드를 오가야 하는 불편함이 생깁니다.
이러한 문제를 해결하기 위해, 생명주기 중심으로 코드를 나누는 대신 훅을 사용해 기능 단위로 컴포넌트를 구성하는 방식이 필요하게 되었습니다.

### 클래스 기반 구조의 부담

클래스 컴포넌트는 자바스크립트 언어 특성과 강하게 결합되어 있습니다. 그 결과 리액트 자체의 개념 이해는 물론 자바스크립트 문법과 실행 방식까지 함께 이해해야 했습니다.

- `this`는 호출 방식에 따라 달라지기 때문에 동작을 예측하기 어렵습니다
- 이벤트 핸들러를 등록할 때 메서드를 직접 바인딩해야 하는 등 추가적인 문법 지식이 필요합니다
- 함수 컴포넌트와 클래스 컴포넌트가 공존하면서 어떤 방식을 언제 사용해야 하는지 명확하지 않았고, 팀마다 다른 스타일이 생기기도 했습니다

또한 클래스는 자바스크립트의 동적인 특성에 의존하기 때문에 도구 친화적이지 않습니다.

- `this`와 프로토타입 기반 구조로 인해 코드의 실행 방식이 런타임에 결정돼 정적 분석이 어렵습니다
- 그 결과 코드 최소화, 트리 셰이킹, 핫 리로딩과 같은 최적화 작업이 복잡해질 수 있습니다

이렇게 클래스 기반 구조는 학습 부담도 높이고 최적화 측면에서도 불리합니다.
이를 해결하기 위해 훅은 클래스 없이도 상태와 생명주기를 사용할 수 있도록 해서, 리액트를 함수 중심 모델로 단순화하고 최적화 친화적인 구조를 지향하게 되었습니다.

## 종류

### State 훅 사용하기

`useState`는 함수 컴포넌트 내부에서 상태를 선언하고 관리할 수 있게 해주는 훅입니다.

```js
import { useState } from "react";

function Example() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  );
}
```

`useState`는 두 개의 값을 반환합니다.

```js
const [count, setCount] = useState(0);
```

위 예에서 `count`는 현재 상태 값, `setCount`는 상태를 변경하는 함수입니다.
0은 초깃값이며, 버튼을 클릭해 `setCount`를 호출하면 리액트는 새로운 상태를 저장하고 컴포넌트를 리렌더링합니다. 그 결과 변경된 `count` 값이 화면에 반영됩니다.<br>
`useState`는 `[현재 상태, 상태 변경 함수]` 형태의 배열을 반환하며, 흔히 이를 구조 분해해 사용합니다.
